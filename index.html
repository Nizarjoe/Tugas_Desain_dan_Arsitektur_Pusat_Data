Deskripsi Singkat (Narasi Teknis)

Masalah komputasi konvensional: beban pengelolaan infrastruktur, keterbatasan elastisitas, dan biaya awal tinggi untuk penyediaan server fisik. Solusi cloud computing: menyediakan abstraksi sumber daya (compute, networking, storage) yang elastis, tersedia melalui layanan terkelola (PaaS), memungkinkan fokus pada aplikasi dan data. Relevansi dengan pusat data modern: pusat data modern mengandalkan virtualisasi, orkestrasi, dan CDN untuk performa global serta model tanggung jawab bersama antara penyedia cloud dan pengguna.

Arsitektur Pusat Data Cloud

User
Pengguna akhir (browser) mengakses aplikasi melalui internet.
Internet
Jalur publik yang membawa request ke CDN / hosting.
CDN + SSL (GitHub Pages)
Konten statis terdistribusi; TLS/HTTPS aktif untuk enkripsi end-to-end.
PaaS Hosting Environment
Platform terkelola (GitHub Pages sebagai PaaS level statis) mengabstraksi infrastruktur fisik.
Client-side Data Processing
Semua logika analitik berjalan di browser (no backend), data simulasi & pemrosesan diselesaikan client-side.
Visualization Layer (Chart.js)
Visualisasi interaktif untuk monitoring, berbasis library JS yang dimuat via CDN.
Konsep- konsep penting
  • Availability: CDN + PaaS meminimalkan downtime untuk konten statis; multiple edge node meningkatkan ketersediaan.
  • Scalability: Layanan PaaS dan CDN secara transparan menskalakan distribusi konten sesuai trafik.
  • Fault tolerance: Replikasi di edge nodes dan caching mengurangi dampak kegagalan pusat data tunggal.
  • Resource abstraction: PaaS menyembunyikan detail hardware; developer berfokus pada aplikasi dan data.

Data Analytics Dashboard (Simulasi)

Dashboard berikut mensimulasikan metrik yang biasa dimonitor pada pusat data cloud: traffic, penggunaan resource, dan distribusi beban. Semua data dihasilkan dan diproses di sisi client (browser).

Line Chart — Traffic Cloud (Requests per menit)

Makna: Pola trafik menunjukkan fluktuasi permintaan client. Lonjakan dapat menandakan event, deployment, atau serangan DDoS. Monitoring membantu autoscaling dan kapasitas CDN.

Bar Chart — Resource Usage (Simulasi CPU, Memory, I/O)

Makna: Perbandingan penggunaan sumber daya membantu mengidentifikasi bottleneck (mis. CPU-bound vs memory-bound). Dalam lingkungan PaaS, metrik ini membantu penyedia menentukan limits dan quotas.

Pie Chart — Load Distribution (Region / Service)

Makna: Distribusi beban antar region atau service menunjukkan di mana replikasi dan caching perlu dioptimalkan untuk latency dan cost efficiency.

Kontrol Simulasi

Gunakan tombol untuk mereset atau randomize data simulasi berikut untuk demo cepat.

Keamanan Cloud

HTTPS / SSL

Transport Layer Security (TLS) melindungi integritas dan kerahasiaan data antara pengguna dan CDN/hosting. GitHub Pages menyediakan sertifikat TLS otomatis untuk domain github.io dan custom domain via Let's Encrypt — memastikan enkripsi end-to-end.

Isolasi Platform & Tidak adanya exposed backend

Aplikasi ini adalah statis (front-end-only). Tidak ada endpoint backend yang terekspos — mengurangi permukaan serangan. PaaS dan CDN memberikan lapisan isolasi dari hardware fisik dan layanan lain di multi-tenant environment.

Shared Responsibility Model

Dalam model tanggung jawab bersama, penyedia cloud (GitHub Pages/CDN) bertanggung jawab atas keamanan infrastruktur fisik, jaringan, dan TLS, sedangkan pengguna bertanggung jawab atas konten aplikasi, konfigurasi DNS/domain, serta kontrol akses ke repositori sumber.
