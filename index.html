Deskripsi Proyek

Proyek ini merupakan implementasi akademis dari arsitektur pusat data modern berbasis cloud computing model PaaS (Platform as a Service). Dalam konteks komputasi konvensional (on-premise), pengelolaan pusat data memerlukan investasi besar dalam hal infrastruktur fisik, maintenance, dan sumber daya manusia yang kompeten.

Melalui transformasi cloud computing, pusat data berkembang dari model fisik terpusat menjadi layanan terdistribusi dengan skalabilitas elastis. Model PaasaS dipilih karena menyediakan platform lengkap untuk pengembangan dan deployment aplikasi tanpa kompleksitas pengelolaan infrastruktur dasar.

Relevansi proyek ini dengan disiplin Sains Data terletak pada implementasi dashboard monitoring, analytics, dan observability yang merepresentasikan bagaimana data center modern mengumpulkan, memproses, dan memvisualisasikan data operasional secara real-time untuk pengambilan keputusan berbasis data.

Validitas Arsitektur Cloud

Arsitektur ini valid sebagai arsitektur cloud karena memenuhi karakteristik NIST: On-demand self-service, broad network access, resource pooling, rapid elasticity, dan measured service. GitHub Pages sebagai platform PaaS menyediakan semua layanan infrastruktur secara abstrak.

Keamanan Cloud

Implementasi keamanan cloud dalam arsitektur ini mengikuti Shared Responsibility Model yang merupakan paradigma fundamental dalam cloud computing.

Enkripsi Data In-Transit

GitHub Pages menyediakan HTTPS/SSL secara default, mengenkripsi semua data yang ditransmisikan antara user dan platform cloud. Ini melindungi dari serangan man-in-the-middle (MITM).

Terimplementasi: HTTPS aktif dengan sertifikat SSL/TLS
Isolasi Platform PaaS

Pada model PaaS, penyedia cloud (GitHub) mengisolasi lingkungan eksekusi tiap pengguna, mencegah interferensi antar aplikasi (noisy neighbor) dan meningkatkan keamanan multi-tenant.

Terimplementasi: Isolasi melalui GitHub Pages
Mitigasi Risiko Backend

Dengan arsitektur 100% client-side, tidak ada backend yang terbuka sehingga mengurangi surface area attack. Semua pemrosesan terjadi di browser pengguna.

Terimplementasi: Tidak ada backend yang terekspos
Shared Responsibility Model

Dalam model ini, GitHub bertanggung jawab untuk keamanan of the cloud (infrastruktur), sementara pengembang bertanggung jawab untuk keamanan in the cloud (aplikasi dan data).

Terimplementasi: Pembagian tanggung jawab keamanan
